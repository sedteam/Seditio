# Глава 12. Персональная файловая система (PFS) и обработка медиа

В этой главе рассматривается Персональная файловая система (PFS) Seditio CMS, структура баз данных для файлов и папок, права доступа к файловому пространству (ACL) и дисковые лимиты, а также низкоуровневый и высокоуровневый программный интерфейс обработки медиа-ресурсов (Image API, ресайз на лету, наложение водяных знаков и асинхронная Drag & Drop загрузка).

---

## 12.1. Архитектура PFS на уровне Базы Данных

Персональная файловая система (PFS) является встроенным модулем ядра Seditio. Схемы таблиц PFS регистрируются инсталлятором при активации модуля через файл [pfs.install.php](../modules/pfs/pfs.install.php).

Данные PFS хранятся в двух связанных системных таблицах базы данных: `sed_pfs` (файлы) и `sed_pfs_folders` (виртуальные папки).

### 12.1.1. Таблица загруженных файлов (`sed_pfs`)
Каждый файл, загруженный пользователем на сайт через PFS, описывается записью в этой таблице:

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| **`pfs_id`** | `int` | Уникальный первичный ключ записи (AUTO_INCREMENT). |
| **`pfs_userid`** | `int` | ID пользователя-владельца файла (внешний ключ на `sed_users`). |
| **`pfs_date`** | `int` | UNIX-время загрузки файла. |
| **`pfs_file`** | `varchar(255)` | Физическое имя файла на сервере (хранится в каталоге `/datas/users/`). |
| **`pfs_extension`** | `varchar(8)` | Расширение файла в нижнем регистре (например, `jpg`, `zip`, `pdf`). |
| **`pfs_folderid`** | `int` | ID виртуальной папки, к которой принадлежит файл (внешний ключ на `sed_pfs_folders`). Если файл загружен в корень, равен `0`. |
| **`pfs_title`** | `varchar(255)` | Название файла, заданное пользователем при загрузке. |
| **`pfs_desc`** | `text` | Текстовое описание файла. |
| **`pfs_size`** | `int` | Размер файла в байтах. |
| **`pfs_count`** | `int` | Счетчик обращений (скачиваний/просмотров) файла на сайте. |

### 12.1.2. Таблица виртуальных папок (`sed_pfs_folders`)
Пользователи могут группировать свои файлы по виртуальным каталогам. Структура папок описывается следующей схемой:

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| **`pff_id`** | `int` | Первичный ключ папки (AUTO_INCREMENT). |
| **`pff_userid`** | `int` | ID пользователя-владельца папки. |
| **`pff_parentid`** | `int` | ID родительской папки (для поддержки вложенности). По умолчанию равен `0`. |
| **`pff_date`** | `int` | UNIX-время создания папки. |
| **`pff_updated`** | `int` | UNIX-время последнего обновления содержимого папки. |
| **`pff_title`** | `varchar(255)` | Название папки. |
| **`pff_desc`** | `text` | Описание папки. |
| **`pff_type`** | `tinyint` | Флаг приватности папки (`0` — публичный доступ, `1` — только для владельца). |
| **`pff_sample`** | `int` | ID файла (`pfs_id`) из этой папки, который служит превью-картинкой (обложкой) папки. |
| **`pff_count`** | `int` | Счетчик файлов, содержащихся в данной папке. |

## 12.2. Права доступа (ACL) и квоты дискового пространства

PFS является контролируемым ресурсом, доступ к которому жестко регламентируется правами ACL и дисковыми квотами групп пользователей.

### 12.2.1. Контроль доступа (ACL)
Доступ к модулю PFS определяется правами авторизации для кода ресурса `pfs` (опция `a`):
* **Гости** (`SED_GROUP_GUESTS`) по умолчанию лишены каких-либо прав (запрещены чтение и запись).
* **Пользователи** (`SED_GROUP_MEMBERS`) имеют права `RW` (Чтение и Запись собственных файлов). Модифицировать чужие файлы пользователи не могут.
* **Администраторы** (`SED_GROUP_SUPERADMINS`) имеют права `RWA12345` (полный доступ, включая право просмотра и удаления файлов любых пользователей через Панель управления).

### 12.2.2. Лимиты и дисковые квоты групп
Для предотвращения переполнения диска сервера в настройках каждой группы пользователей (вкладка «Права/Группы» в админ-панели) задаются индивидуальные дисковые квоты:
1. **`grp_pfs_maxfile`** — максимальный допустимый размер одного загружаемого файла (в КБ). Если файл превышает этот объем, ядро блокирует загрузку.
2. **`grp_pfs_maxtotal`** — максимальный суммарный объем файлов (квота дискового пространства на одного пользователя группы в КБ).

При каждой попытке загрузки файла ядро Seditio суммирует размер всех уже загруженных файлов пользователя в БД:
```sql
SELECT SUM(pfs_size) FROM sed_pfs WHERE pfs_userid = $userid
```
И сопоставляет сумму вместе с размером нового файла с лимитом `grp_pfs_maxtotal`. При превышении лимита загрузка прерывается ошибкой безопасности.

## 12.3. Встроенная обработка изображений (Image API)

Все функции работы с изображениями сосредоточены в системном файле `system/functions.image.php`. Seditio поддерживает работу как с расширением GD, так и с ImageMagick (выбор настраивается административно).

### 12.3.1. Основные функции ресайза и обрезки
Для удобного применения непосредственно внутри HTML-шаблонов в Seditio CMS реализованы две высокоуровневые функции-фильтра:

1. **`resize_image($filename, $width, $height, $set_watermark, $use_webp)`**
   Масштабирует изображение до указанных ширины и высоты с сохранением исходных пропорций (картинка вписывается в заданные рамки).
2. **`crop_image($filename, $width, $height, $set_watermark, $use_webp)`**
   Обрезает изображение точно под указанные пропорции. Картинка центрируется, лишние края отсекаются.

**Параметры функций:**
* `$filename` (string) — относительный путь к исходному файлу на диске (например, `datas/users/photo.jpg`).
* `$width` (int) — целевая ширина в пикселях.
* `$height` (int) — целевая высота в пикселях.
* `$set_watermark` (bool, необязательный) — флаг наложения водяного знака.
* `$use_webp` (bool, необязательный) — флаг конвертации изображения в формат WebP для снижения веса.

### 12.3.2. Концепция отложенного ресайза
Для того чтобы генерация тяжелых изображений не замедляла загрузку основных HTML-страниц сайта, в Seditio применен механизм **отложенного ресайза**:
1. При рендеринге страницы шаблонизатор вызывает функцию `resize_image()` или `crop_image()`.
2. Функция не производит ресайз физически. Она лишь кодирует параметры сжатия (ширину, высоту, режим сжатия) и возвращает виртуальный путь в директорию кэша `$cfg['res_dir']` (по умолчанию `/datas/resized/`):
   *Пример возвращаемого пути:* `datas/resized/photo.resize.1920.1080.jpg`.
3. Браузер пользователя получает готовый HTML-код с этим виртуальным путем и отправляет GET-запрос к изображению.
4. Сервер перенаправляет запросы несуществующих файлов из папки `/datas/resized/` на скрипт-обработчик (модуль `resizer`).
5. Модуль `resizer` перехватывает запрос, парсит имя файла, находит оригинал, выполняет его физический ресайз с помощью функции `sed_resize()`, сохраняет готовый файл в кэш и отдает браузеру.
6. При повторных запросах веб-сервер отдает уже готовый сжатый файл с диска напрямую, минуя PHP.

### 12.3.3. Наложение водяных знаков (`sed_image_merge`)
Если при ресайзе передан флаг `$set_watermark = true`, система вызывает функцию `sed_image_merge()`.
* Настройки водяного знака (путь к PNG-файлу с прозрачностью, прозрачность в процентах и позиционирование) считываются из конфигурации ядра.
* Слияние изображений происходит с сохранением альфа-канала с помощью функции `sed_imagecopymerge_alpha()`, предотвращающей появление артефактов на стыках полупрозрачности водяного знака.

## 12.4. Drag & Drop асинхронная загрузка файлов

Для удобной пакетной загрузки медиа-ресурсов Seditio CMS оснащена встроенным асинхронным загрузчиком, функции которого объявлены в `system/functions.image.php`.

### 12.4.1. Ключевые функции загрузчика
* **`sed_image_upload_html($opts)`** — генерирует HTML-разметку области загрузки (Drag & Drop дроп-зону), автоматически подключает необходимые CSS/JS-библиотеки и регистрирует события.
  Принимает ассоциативный массив настроек со следующими ключами:
  * `'prefix'` (string) — префикс имен полей (по умолчанию `'images'`).
  * `'title'` (string) — человекочитаемый заголовок дроп-зоны.
  * `'max_files'` (int) — лимит загружаемых файлов (по умолчанию `0` — без ограничений).
  * `'sortable'` (bool) — поддержка перетаскивания и сортировки загруженных картинок (по умолчанию `true`).
  * `'dropzone'` (bool) — включение Drag & Drop области (по умолчанию `true`).
  * `'url_upload'` (bool) — разрешение загрузки картинок по внешним ссылкам (по умолчанию `true`).
  * `'existing'` (array) — массив уже загруженных ранее файлов для их отображения.
  * `'id'` (string) — уникальный DOM ID элемента.
  * `'accept'` (string) — типы разрешенных файлов (по умолчанию формируется автоматически на основе настроек GD/ImageMagick).
* **`sed_image_upload_save($tmp_path, $dest_path, $max_w, $max_h, $mode = 'crop', $is_uploaded = true)`** — производит низкоуровневую обработку (ресайз/обрезку) и безопасное сохранение файла на диске.
* **`sed_image_upload_process($opts)`** — обрабатывает асинхронный POST-запрос с файлами, проверяет права доступа пользователя к PFS и дисковые квоты, сохраняет оригиналы файлов и автоматически создает их эскизы в PFS.

### 12.4.2. Использование Drag & Drop загрузчика в плагинах
Для интеграции загрузчика в форму создания контента разработчику плагина достаточно вызвать хелпер в PHP-контроллере:
```php
// Генерируем HTML-код дроп-зоны для поля с префиксом 'album_images'
$uploader_html = sed_image_upload_html(array(
    'prefix' => 'album_images',
    'max_files' => 10
));

$t->assign('MYPLUGIN_UPLOADER', $uploader_html);
```
И вывести тег в TPL-шаблоне:
```html
<div class="form-group">
    <label>Загрузить фотографии альбома:</label>
    {MYPLUGIN_UPLOADER}
</div>
```
При отправке формы ядро автоматически обработает переданные файлы, привяжет их к текущему сеансу PFS пользователя и сохранит пути в базу данных.
